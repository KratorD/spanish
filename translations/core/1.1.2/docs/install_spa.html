Zikula: La versión adam_baum

Zikula es un framework de aplicaciones software libre, para sitios Web dinámicos. El núcleo del sistema incluye un API completo sobre el cual desarrollos de terceras partes pueden ser construidos. Zikula tiene una comunidad muy fuerte, la cual provee una gran variedad de extensiones para el sistema y un centro de soporte en los Foros de Zikula y en la Comunidad hispana de Zikula.

Le damos la bienvenida las contribuciones de cualquiera, mira Cómo puedes ayudar.

2. Consideraciones del servidor

Antes de comenzar tu actualización, hay una guía y algunos requerimientos que debes considerar para hacer la actualización, instalación, ejecución, y aseguramiento de Zikula algo fál. Por favor lee el codumento de consideraciones respecto a software.

3. Nueva Instalación

Hay dos métodos de instalación para Zikula:

  • Instalador gráfico vía Web
  • Importación manual del volcado SQL

Instalación gráfica

Para usar el instalador, simplemente ejecuta el script install.php en la raiz de tu sitio Zikula en tu navegador, ej. http://www.ejemplo.com/install.php. Si quieres instalar Zikula en una subcarpeta 'foo' la URL será http://www.ejemplo.com/foo/install.php.

El instalador te provee instrucciones en el camino. Zikula requiere una base de datos y puedes ahorrar problemas si la creas antes de correr el script de instalación. Aunque es posible que el instalador de Zikula trate de crear la base de datos, esto no siempre será posible pues requerirás altos niveles de acceso a la base de datos. En la mayoría de los hostings esto no siempre está disponible así que debes crear la base de datos a través del panel de control administrativo. Si haces esto, anota el nombre de la base de datos, nombre de usuario y contraseña para usarlo en el instalador.

Una vez que el proceso de instalación se ha completado, puedes visitar tu sitio Zikula abriendo el index.php en tu navegador, ej. http://www.ejemplo.com/index.php. En la gran mayoría de casos, esta se abrirá automáticamente cuando escribas http://www.ejemplo.com en tu navegador.

Importación manual del volcado SQL (No recomendado)

Si deseas importar el volcado SQL para instalar Zikula manualmente, lo puedes acer a través de una herramienta como phpMyAdmin. El volcado proporcionado es de una base de datos MySQL 5.x, y pueda que no funcione con otras bases de datos, o versiones de MySQL. Una vez hayas importado el volcado SQL, deberás editar manualmente el archivo config/config.php y añadir los detalles correspondientes de la conexión a la base de datos el el área reservada para la conexión por 'defecto'. Los valores para nombre de usuario y contraseña deberían ser codificados en base64 para seguridad adicional (hay varios codificadores disponibles en Internet). Con esta instalación, el nombre de usuario del administrador es "admin" y la contraseña es "Password" (sin las comillas).

4. Nota final

Las modificaciones al código fuente del núcleo del sistema de PostNuke o a la base de datos no son soportados. Hacer esto puede causar problemas graves cuando quieras actualizar el sistema el el futuro y por lo tanto esos 'hacks' no son recomendados. PostNuke tiene un sistema de extensiones muy flexible y varias formas de personalizar el sistema por eso te recomendamos que consultes acerca de esto en la documentación para desarrolladores.

Esperamos que disfrutes usando esta versión de Zikula.
